# Project Task Result Formats Specification

## Overview
This document specifies the requirements for handling diverse output formats generated by backend AI agents upon task completion. It extends the task execution workflow by defining how the frontend should process and display different types of task results.

## 1. Supported Result Formats
A resolved task can now produce results in several different formats. The frontend application must be equipped to handle and correctly render each of the following:

- **Markdown:** Text-based deliverables. The UI should safely parse and render the markdown content, displaying formatted text, lists, and links.
- **PDF:** Document deliverables. The UI should provide a built-in PDF viewer for inline reading, or at a minimum, offer a clear download/preview link.
- **Video:** Media deliverables. The UI must embed a video player with standard playback controls to view the resulting video file.
- **Image:** Visual deliverables. The UI should display the image directly in the workflow view, potentially with zoom or full-screen capabilities.

## 2. API Integration Updates
The existing endpoint for retrieving task results must be handled dynamically by the frontend:

**Endpoint:** `GET /api/v1/companies/{company_id}/projects/{project_id}/tasks/{task_id}/result`

Depending on the backend implementation, the frontend must determine the format either through standard HTTP `Content-Type` headers OR via a structured JSON response that specifies the `type` of media and its content/URL.

### Error Handling (Validation Errors)
The frontend must correctly handle generic API validation errors. If the request is invalid, the API will return an Unprocessable Entity (HTTP 422) response conforming to the standard FastAPI validation error structure:

```json
{
  "detail": [
    {
      "loc": [
        "string",
        0
      ],
      "msg": "string",
      "type": "string",
      "input": "string",
      "ctx": {}
    }
  ]
}
```
The application should intercept this structure and display the `msg` safely to the user to provide clear feedback on what went wrong during the result retrieval.

## 3. UI Context & Workflow View
- **Result Presentation:** Once a task reaches the "Completed" state, the workflow view must transition to show the final deliverables.
- **Dynamic Component Loading:** The UI must implement a factory or dynamic component loader that instantiates the correct viewer (MarkdownRenderer, PdfViewer, VideoPlayer, ImageViewer) based on the detected format of the task result.
